## Gemini 3.1-pro holistic-adversarial — VERDICT: BLOCK (verbatim findings)

> **[CRITICAL 1] The "Fast-Terminalize on SOURCE ERROR" trap turns partial batch successes into total job failures.** NotebookLM corpus imports frequently experience partial failures (1 malformed PDF out of 100). If the host wait hard-fails the entire pipeline the moment *any* source returns `ERROR=3`, you destroy job completion rates for large research tasks. **Fix:** block ONLY while `PROCESSING(1)+PREPARING(5) > 0`; once all sources resolve to `READY(2)` OR `ERROR(3)`, complete the wait and resume spawn #2 with the partial-success state.
>
> **[MAJOR 2] [C] vs [D] Prompt Conflict (Migration Oversight).** [C] instructs the agent to poll in-turn; [D] requires dispatch-and-yield. If [C]'s relocated poll remains when [D] ships, the agent polls in-turn instead of yielding, bypassing [D]'s host-side wait entirely. **Fix:** state in §3.1/§2[D] that building [D] requires REMOVING/disabling [C]'s relocated poll — clean transition "poll in-turn" → "dispatch and yield".
>
> **[MAJOR 3] Adjudication D-i vs D-ii → D-ii is MANDATORY.** An external import of 100+ sources can take tens of minutes; blocking the worker thread for an hour on an external dependency destroys throughput and risks OS idle timeouts. **Fix:** mandate D-ii (agent yields, worker parks, decoupled sweep owns polling + re-dispatch).
>
> **[MINOR 4] Adjudication Q4 → tighten the gate, not the prompt.** Relying on a prompt to prevent `"complete (pending import)"` violates the doc's own thesis (§1.4). **Fix:** tighten `COMPLETE_AUGMENTED` (`state-evaluation.ts:318`) to reject parenthetical caveats deterministically.
>
> **[MINOR 5] Empty Source List Race Condition.** If the agent writes `awaiting_import` but the Phase-3 dispatch fails, `source list` returns empty → the wait condition (no 1s/5s) evaluates false → host instantly resumes spawn #2 with zero sources → garbage report. **Fix:** assert total source count `> 0` before waiting; short grace for API lag, else fail-closed.
