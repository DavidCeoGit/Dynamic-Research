## 5. Cost Baseline — **ACTION NEEDED**

### Paid API surface
| Path | Provider | Type | Token cap | Auth |
|---|---|---|---|---|
| `POST /api/queue/extract-context` | Anthropic (`@ai-sdk/anthropic`) | `generateObject` claude-sonnet-4 | **NONE** | **NONE** |
| `POST /api/queue/generate-questions` | Anthropic (`@ai-sdk/anthropic`) | `generateObject` claude-sonnet-4 | **NONE** | **NONE** |
| `agent/executor.ts` `claude -p` subprocess | Claude CLI agentic loop (Perplexity MCP, Chrome MCP, WebSearch) | ~$1-3/run | **NONE** | n/a (worker only) |
| `regenerate-studio-products.ts` | NLM local CLI | Quota-bound | n/a | n/a |

### Cost controls present
- PID-file singleton (effectively `MAX_CONCURRENT_JOBS=1`)
- Atomic queue claim via `FOR UPDATE SKIP LOCKED`
- `MAX_JOB_DURATION_MS=5,400,000` (90 min) with SIGTERM/SIGKILL escalation; tracks active-only ms (Windows S3/S4 sleep aware)
- `DRY_RUN=true` worker kill switch
- `pipeline_mode='studio_only'` cheap path + 400-guard against silent downgrade

### Cost controls MISSING
- **No `maxOutputTokens` on either unauth Anthropic route.** `generateObject` runs to schema completion at SDK default.
- No daily / monthly spend ceiling anywhere (`DAILY_BUDGET_USD`, `MONTHLY_CAP`).
- No throttle queue (no analog of GravityClaw's `AutonomousThrottle max=2/queue=5`).
- No reasoning kill switch covering frontend routes (`DRY_RUN` only gates worker).
- No rate limit on `POST /api/queue` itself — a bot can flood pending rows.

### Telemetry — ABSENT
Zero hits for `cost`, `tokens.usage`, `prometheus`, `grafana`. Claude CLI is spawned with `--output-format json` but the worker `slice(0, 200)` logs stdout and discards token totals. No `research_usage` table.

### Dollar-risk scenarios
1. Wedged worker → ≤ ~$3-5 (singleton + 90-min cap). **Bounded.**
2. Bot floods `POST /api/queue` → ~$50-80/day at 28 jobs/day worst case. Bounded by serial worker.
3. **Bot hammers `POST /api/queue/extract-context` or `generate-questions`** → no token cap, no rate limit, no auth, ~$3/M output × ~2k tokens × 10 req/s sustained = **~$5,000/day worst case**. **UNBOUNDED**. Highest exposure.

### Recommendations (priority order)
1. **HIGH — cap the two unauth Anthropic routes.** Add `maxOutputTokens` (suggested 1500 generate-questions, 2000 extract-context) + Vercel Edge rate-limit (per-IP, e.g. 20/hr) or Turnstile origin check. Closes the $5K/day hole. **Highest-leverage single fix in this review.**
2. **HIGH — cost telemetry.** Parse Claude CLI's `--output-format json` final summary (already in spawn args) and write `{job_id, input_tokens, output_tokens, cache_read_tokens, est_usd, duration_ms}` to a new `research_usage` Supabase table.
3. **MEDIUM** — `DAILY_RUN_CAP` env var; worker refuses to claim if count(today's runs) > cap. Mirrors GravityClaw spirit.
4. **MEDIUM** — `--max-turns N` on spawned `claude -p`. Bounds runaway agentic loops within the 90-min window.
5. **MEDIUM** — Turnstile or verified-email gating on `POST /api/queue`.
6. **LOW** — Perplexity call meter.
7. **LOW** — alert on 3+ consecutive `failed` rows.
